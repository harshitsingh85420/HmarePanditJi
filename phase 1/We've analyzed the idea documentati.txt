### **PROMPT 3.2 â€“ TRAVEL CALCULATION SERVICE**
```
Implement the travel calculation service at `services/api/src/services/travel.service.ts`. This powers the travel mode tabs and booking wizard. In Phase 1, all calculations are based on the hardcoded `CityDistance` table and fixed fare tables (no live API calls).

**Methods:**
- `getDistance(fromCity, toCity)`: query `CityDistance` table (bidirectional). Return `{ distanceKm, estimatedDriveHours }` or null if not found.
- `calculateSelfDrive(distanceKm, eventDays, foodArrangement)`:

  * drivingCost = roundTrip * 12
  * travelDays = ceil(distanceKm / 400) * 2  (max 400km/day)
  * foodAllowanceDays = travelDays + (foodArrangement === "PLATFORM_ALLOWANCE" ? eventDays : 0)

  * totalTravelCost = drivingCost + foodAllowance
  * travelServiceFee = totalTravelCost * 0.05, GST = travelServiceFee * 0.18
  * Return breakdown and totals.
- `calculateTrain(distanceKm, ...)`: use distance bands (0-200,200-500,500-1000,1000-2000) with fixed 3AC fares (e.g., â‚¹500, â‚¹1200, â‚¹2500, â‚¹3500). Multiply by 2 for round trip. Add local cab cost (â‚¹1600 fixed). Compute food allowance similarly.

- `calculateCab(distanceKm, ...)`: for distances <300km, cab fare = distance * 15 * 2, plus food allowance.
- `calculateAllOptions(fromCity, toCity, eventDays, foodArrangement, panditPreferences)`: returns array of possible modes with their cost breakdowns, sorted by total cost.

**Controller (`travel.controller.ts`):**
- `POST /api/travel/calculate` â€“ accepts { fromCity, toCity, travelMode?, eventDays?, foodArrangement? }. If travelMode specified, calculate that mode only; otherwise all options.
```
